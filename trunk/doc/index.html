This project is in an early state. I suppose, it's not yet satble enough, to build a file-release. The planned steps before building a file-release are:
  1. Write some more unit-tests and make them running without errors:
    • Check, if the priority-feature works. Rules can have priorities, so that you have more control over the order in which rules are fired.
    • Check, if the start-hash feature works. This feature allows one to partition the domain of all possible facts. This can boost performance, when inserting or removing facts into/from a context.
  2. Do some malloc-debugging to eliminate potential memory leaks.
Those, who wanna try it, can check out the main development branch directly from the Subversion repository:
   svn checkout svn://svn.berlios.de/maren/trunk maren
For more information go to the project's overview at berliOS.
